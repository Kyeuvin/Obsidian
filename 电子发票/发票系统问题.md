# 发票系统

## 1. 关于公司抬头

- [x] 1. 发票查询时代理无法选择公司抬头，由系统识别登录账户的抬头。  

- [ ] 2. 发票开具时代理无法修改抬头，自动填入汇款方的公司抬头。    

- [x] 3. 开具主体绑死汇款的地方（商务、旅行社等）。要保证开发票的公司和我们签过合同。

## 2. 发票备注

- [x] 1. 不论是否填写了发票备注的要转到后台给财务审核。输入了备注给代理提示：添加备注需要审核，会延长开票时间。

- [ ] 2. 可以给代理提供一键填入的常用格式，如备注票号、旅客姓名等。放在开票备注的输入框旁边，点击自动填入。

- [x] 3. 发票备注不大于200字。

- [x] 4. 前台中给代理填写的备注内容我们放在数据库中的“需求备注”字段。而真实要提交的内容我们放在“开票备注字段”，这个字段的数据是经过财务审核后的备注。

- [x] 5. 发票提交后的状态是待审核，后台财务审核后需要支付的发票状态会变成待支付。而不需支付的发票会跳过待支付进行开票。

## 3. 关于金额

- [x] 1. 发票上的开票金额（含税）是含税票价，代理不能修改，我们可以修改。

- [x] 2. 税点在开具特定业务类型的发票时才包含（如改期费），值为开票金额的6%。

- [x] 3. 不收10块的额外费用，代理一般只需要支付税点。

- [ ] 4. 在电子发票查询时只有有税点的发票会显示支付按钮，不需支付的直接跳到下一步。

## 4. 开具的限制

- [x] 1. 一次性最多同时开100张发票。

- [ ] 2. 国内航班只在航后可开。
    
    国际航班如果是一张一张开，则可以航前开；如果要打包一起开，则必须航后开。

- [ ] 3. 发票申请打包一起开的时候，必须保证业务类型一致，公司抬头一致。

- [x] 4. 只给代理提供出票日期一年内的发票开具。

## 5. 其他

- [ ] 1. 查询中execl导入可以只查询票号。

- [x] 2. 业务类型对不同的用户进行不同的显示，请整理并提供给我们。

- [x] 3. 发票内容的种类也请提供给我们。

- [ ] 4. 同一票号的发票金额加起来不超过实际的开票金额（含税）
    
    如一张机票的开票金额是2417元，其中票价2387元，服务费30元，开了两张发票。则这两张发票的开票金额不能超过2417。
